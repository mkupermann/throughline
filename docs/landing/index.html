Everything Claude Code was missing
Six capabilities, one local database, zero recurring costs.
💾

JSONL Session Ingestion

Reads every JSONL file from ~/.claude/projects/, deduplicates by SHA-256, and stores full message history, tool calls, and token counts in PostgreSQL.

🧠

AI Memory Extraction

Sends conversation windows through Claude and stores typed memory chunks: decisions, patterns, insights, error solutions, contacts, preferences, and workflows. Two backends: the Anthropic API or the Claude Code CLI in headless mode.

🔍

Semantic Search

Cosine similarity over 1536-dim OpenAI or 768-dim Ollama embeddings, indexed with HNSW for fast approximate nearest-neighbor lookups. Works fully offline with Ollama.

🌐

Temporal Knowledge Graph

Entities (people, projects, technologies) and typed relationships tracked across sessions with valid_from / valid_until. Interactive visualization in the GUI.

🔗

Context Pre-Loader Hook

A SessionStart hook queries the DB for the current project and injects a concise memory summary before Claude's first response — every new session starts informed.

🔅

Self-Reflecting Memory

A periodic reflection pass merges near-duplicates, flags contradictions, supersedes outdated decisions, and logs every action. Memory stays clean without manual curation.

Up in one command

Docker Compose brings up PostgreSQL 16 + pgvector + the Streamlit GUI. Your ~/.claude directory is mounted read-only so ingestion scripts can see your sessions immediately.

  1. Clone the repo and start all services with Docker Compose.
  2. Open the Streamlit GUI in your browser at port 8501.
  3. Run the ingest script to populate the database from your existing sessions.
  4. Optionally enable Ollama for fully offline local embeddings.
Full installation guide 
# Clone and start all services
git clone https://github.com/mkupermann/throughline.git
cd throughline
docker compose up -d

# Open the GUI
open http://localhost:8501

# Ingest your existing Claude sessions
docker compose exec gui \
  python3 scripts/ingest_sessions.py

# Optional: local embeddings via Ollama
docker compose --profile embeddings up -d
docker compose exec ollama \
  ollama pull nomic-embed-text
How it works
A closed loop between Claude Code and your memory database.
Step 01

Write

Claude Code writes JSONL session files to ~/.claude/projects/ as you work.

Step 02

Ingest

Hourly launchd job reads new files, deduplicates by hash, and inserts conversations and messages.

Step 03

Extract

Daily job sends conversation windows to Claude; structured memory chunks are stored and embedded.

Step 04

Reflect

Reflection pass deduplicates, resolves contradictions, and supersedes outdated chunks automatically.

Step 05

Recall

At session start, relevant memories are injected. The skill answers in-session queries instantly.
